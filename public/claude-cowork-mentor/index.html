Recipe · 01

CLAUDE.md — תשתית הליבה של Claude Code

הקובץ שמכתיב לכל סשן של Claude Code איך לחשוב, מה לעשות, ומה לא לעשות. רוב הצוותים כותבים אותו רע.

🚫 איך רוב הצוותים כותבים

# My Project # Setup npm install npm run dev # Description This is a React app for users to manage their tasks. # Be helpful and write good code
מה רע פה: מסביר מה עושה הקוד (שה-AI יכול לקרוא לבד). לא נותן הקשר מקצועי, כללי טון, ומה אסור לעשות. ה-AI מנחש.

✅ איך אני כותב

# Domain מערכת ניהול משימות לצוותי פיתוח. משתמשים = developers. # Style — חובה - TypeScript strict - אין any. אין as unknown - שמות בעברית בטקסט, אנגלית בקוד # Architecture rules - DB queries דרך repo/ בלבד - אין SQL ב-route handlers - כל mutation = audit log # What NOT to do - אל תוסיף ספריות חדשות בלי לשאול - אל תכתוב comments מיותרים
למה זה עובד: נותן קונטקסט עסקי, סטנדרטים, ואיסורים. ה-AI לא צריך לנחש — הוא יודע איך אתם רוצים. שינוי דרמטי בפלט.
Recipe · 02

Hooks — אוטומציה שמונעת באגים

פיצ'ר ש-90% מהמשתמשים לא יודעים שקיים. Hooks רצים אוטומטית לפני/אחרי קריאות כלי. מקטין באגים ב-80%.

⚠️ ללא Hooks

# Claude כותב קוד src/api/users.ts ✓ נכתב # 30 דקות אחרי... $ npm run build TS2345: Type 'string' is not assignable to type 'number' in users.ts:42 # 15 דקות אחרי... $ npm test FAIL users.test.ts TypeError: undefined
הבעיה: הבאגים מתגלים מאוחר. כל סיבוב = אובדן זמן וקונטקסט.

✅ עם Hook אחרי כל Edit

{ "hooks": { "PostToolUse": [{ "matcher": "Edit|Write", "hooks": [{ "type": "command", "command": "npm run typecheck" }] }] } } # Claude מקבל typecheck error מיד # ומתקן ב-loop, לפני שעוברים הלאה
למה זה משנה: ה-AI מקבל פידבק מיידי, מתקן בלולאה אותה. הקוד שמגיע אליך — כבר עובר build & tests. פחות סיבובים, פחות באגים.
Recipe · 03

Sub-Agents — עבודה במקביל

במקום Claude אחד שעובד טורית — כמה agents שעובדים במקביל על משימות עצמאיות. דקות במקום שעות.

🐢 טורי (איך רובם עובדים)

# משימה: לבדוק 3 דברים # בקוד 1. Claude קורא file A → 30s 2. Claude קורא file B → 30s 3. Claude קורא file C → 30s 4. Claude מחבר → 20s ───────────────────────── סה"כ: ~110 שניות # משימה: לחפש 5 סוגי באגים 1+2+3+4+5 → 5 דקות חלון הקונטקסט מתמלא, ה-AI מאבד פוקוס
הבעיה: ה-AI מבזבז קונטקסט על קריאות-עזר ומאבד פוקוס במשימות ארוכות.

⚡ מקבילי (איך אני עובד)

# משימה: לבדוק 3 דברים parallel: - Agent("בדוק את A") ┐ - Agent("בדוק את B") ├─ 30s - Agent("בדוק את C") ┘ → merge results: 10s ───────────────────────── סה"כ: ~40 שניות # בנוסף: - כל agent בקונטקסט נקי - האב נשאר עם פוקוס מלא - תוצאות תמציתיות חוזרות
למה זה מנצח: פי 3 מהר ב-3 משימות, פי 10 ב-10 משימות. בנוסף — האב לא רואה את הזבל של כל סוכן, רק את הסיכום. פוקוס נשמר, מהירות עולה.
Recipe · 04

MCP Servers — לחבר את Claude לעולם

Model Context Protocol = איך Claude מתחבר ל-DB שלך, ל-Slack, ל-Linear, ל-Jira, לכל מערכת. בלי לכתוב glue code.

🛠️ MCP servers שאני מתקין רוטינית

{ "mcpServers": { "postgres": { "command": "npx", "args": ["@mcp/postgres"] }, "linear": { "command": "npx", "args": ["@mcp/linear"] }, "playwright": { ... }, "firebase": { ... }, "sentry": { ... } } }
תוצאה: Claude יכול לבצע SELECT על ה-DB, לפתוח tickets, לבדוק errors, להריץ tests בדפדפן — מבלי שתכתוב שורת glue.

💬 שיחה אמיתית עם MCP

הקלילות הזאת = תרגום מיידי בין דיבור לפעולה. בלי zapier, בלי n8n flows, בלי קוד. זה ה-game changer האמיתי של 2026.
Recipe · 05

Code Review של מה ש־AI כותב

הסכנה הכי גדולה ב-AI coding: קוד שעובד, אבל מסתיר חורי אבטחה / queries איטיים / edge cases שלא נחשבו. ככה אני סורק.

🔍 מה לחפש — Checklist

SECURITY □ SQL injection (template strings ב-queries?) □ XSS (innerHTML, dangerouslySetInnerHTML) □ Auth — האם כל endpoint מוגן? □ Secrets ב-git? .env בעריכה? DATA □ Race conditions ב-mutations □ Missing indexes ל-queries תכופים □ N+1 queries (בלולאה async) □ Pagination — תמיד, גם ב-internal EDGE CASES □ Empty arrays, null, undefined □ Network failures, timeouts □ User אם זה לא הוא? □ מה קורה במקביל? QUALITY □ Dead code שה-AI הוסיף "ליתר ביטחון" □ Try/catch גנרי שמחביא בעיות □ ספריות לא נחוצות שנוספו
אני עובר על הצ'ק־ליסט הזה על כל PR שה-AI מייצר. רוב הבאגים נתפסים כאן, לפני שיוצאים מהמחשב.

🎯 דוגמה — באג שתפסתי השבוע

// AI כתב: const user = await db.user.findUnique({ where: { email } }); if (user.password === input) { return generateToken(user); } // 🚨 3 בעיות: // 1. compare ב-=== של hash = // timing attack // 2. אם user == null → crash // 3. אין rate limiting // תיקנתי: const user = await db.user.findUnique({ where: { email } }); if (!user) return rateLimit.fail(); const ok = await bcrypt.compare( input, user.passwordHash ); if (!ok) return rateLimit.fail(); return generateToken(user);
זה ההבדל בין "AI שכותב קוד" ל-"AI שכותב קוד טוב". הוא צריך עין בכירה מעליו. זה התפקיד שלי בליווי.
