💡 למה אני? בגלל השילוב הנדיר

רוב המפתחים שמכירים Supabase לא מבינים פיננסים. רוב מי שמבין חשבונאות לא יודע לכתוב RLS policies. אני מביא את שני העולמות — ניסיון טכני עמוק עם Supabase ו-PostgreSQL, יחד עם ניסיון מעשי בבניית מערכות פיננסיות, חיובים, חשבוניות, ודוחות.

ובנוסף — אני משתמש ב-AI כמכפיל כוח בפיתוח. בדיקת אבטחה שלוקחת ימים? אני עושה את זה בשעות. עם דיוק של סניור.

Supabase עמוק
RLS + Edge Functions		 עובד עם Supabase בפרודקשן. כתבתי RLS policies מורכבות עם הפרדת נתונים בין לקוחות, הרשאות לפי תפקיד, ובעלות על שורות. Edge Functions ב-Deno עם טיפול בשגיאות והגבלת קצב. מבין את הארכיטקטורה מבפנים. ✓
React + TypeScript + Tailwind זה הסטאק היומיומי שלי. 50+ מוצרים ב-React. עבודה ב-TypeScript strict mode. Tailwind עם ספריית קומפוננטות מותאמת. אופטימיזציית ביצועים, טעינה עצלה, פיצול קוד. ✓
Security Audit
תיקוף RLS		 ביצעתי בדיקות אבטחה למערכות SaaS. מבין OWASP Top 10, מניעת SQL injection ו-XSS. יודע לזהות פרצות ב-RLS, הסלמת הרשאות, ודליפת נתונים בין לקוחות. כותב בדיקות אוטומטיות לוולידציית מדיניות. ✓
PostgreSQL מורכב +15 שנים עם Postgres. אופטימיזציית שאילתות, אינדקסים, materialized views, טריגרים, ופרוצדורות. ניסיון עם JSONB, חיפוש טקסט מלא, ופרטישנינג. ✓
OCR + AI
עיבוד מסמכים אוטומטי		 בניתי מערכות OCR עם Tesseract, Google Vision, ו-AI לפרסור מסמכים. ניסיון ספציפי בחשבוניות, קבלות, ודוחות כספיים. הבנת מבנה מסמך, טבלאות, ועברית. ✓
הפקת PDF
מסמכים פיננסיים		 ניסיון עם Puppeteer, jsPDF, ו-react-pdf ליצירת חשבוניות, דוחות, וקבלות. תמיכה מלאה בעברית, לוגואים, טבלאות, וחתימות דיגיטליות. ✓
מערכות פיננסיות
הנהלת חשבונות		 בניתי מערכות חיוב, חשבוניות, ודוחות כספיים. מבין הנהלת חשבונות כפולה, מע"מ, ניכוי מס במקור, ודוחות מס. ניסיון עם Stripe, PayPal, וסליקה ישראלית. ✓
ממשק בעברית מעצב ומפתח ממשקים מימין לשמאל מושלמים. מבין את האתגרים הייחודיים של עברית — טקסט מעורב, מספרים, טבלאות, וטפסים. נגישות מלאה. ✓
