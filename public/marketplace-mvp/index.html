ההצעה

MVP מרקטפלייס שמוכן לייצור אמיתי

200 שעות עבודה זה היקף סביר ל-MVP של מרקטפלייס דו-צדדי — אם הסטאק נכון. אני מציע לבנות את ה-MVP שלכם בקוד אמיתי באמצעות Claude Code, במהירות זהה ל-Bubble, אבל עם תוצאה שתחזיק כשתעלו לסקייל, כשתתחילו לקבל אלפי משתמשים, וכשתבקשו פיצ'רים שאף כלי no-code לא מסוגל לתת.

"בכמה השנים האחרונות ראיתי לא מעט פרויקטים שיצאו לדרך ב-Bubble ולא הגיעו לייצור אמיתי. לא כי המפתחים לא היו טובים — אלא כי הכלי עצמו לא נבנה למה שמרקטפלייס דו-צדדי בזמן אמת צריך. ארכיטקטורת נתונים, Privacy Rules אמיתיים, ביצועים מתחת לסקייל, אינטגרציות סליקה ישראליות — כל אחד מהאלה הופך לסיוט ב-Bubble."

— שלומי זוין, CTO as a Service · 20+ שנות פיתוח

אותה מהירות, איכות אחרת

עם Claude Code בניהול שלי, MVP מרקטפלייס מלא נכנס ל-200 שעות בקלות — בקוד אמיתי, נקי, מתועד. בלי "תקרה של Bubble".

אבטחה אמיתית, לא Privacy Rules

במקום מערכת מבולגנת של Privacy Rules ב-Bubble (שמרקטפלייס מאחד לקריסה מהר), backend אמיתי עם authorization, RLS ב-Postgres, audit logs. בנוי לאמת.

סליקה ישראלית — native

משולם, קארדקום, Tranzila, Pelecard — אינטגרציות native. לא workarounds של Bubble plugin. webhooks, payouts לצד השני, refunds, חיובים מורכבים.

זמן אמת אמיתי

WebSocket / SSE לרילטיים מטצ'ינג, נוטיפיקציות, סטטוסים. לא polling כל 5 שניות שעולה לכם בחבילה של Bubble. ארכיטקטורה נכונה מהיום הראשון.

Mobile-first אמיתי

responsive design ב-CSS modern (flex/grid). לא "Bubble responsive" שנשבר ב-Safari. ביצועים מעולים גם במכשירים חלשים. אפשר אפילו PWA / native לאחר MVP.

הקוד שלכם, ה-IP שלכם

Git repo פרטי שלכם, hosting שלכם, database שלכם. בהתאמה למה שאתם מבקשים ב-NDA וב-IP transfer. בלי vendor lock-in. בלי "מה יקרה אם Bubble יסגרו את החשבון".

מה כלול ב-MVP

שני זרמי משתמש

Onboarding מלא לשני הצדדים — KYC בסיסי, profile, אימות. דיוק מקצועי בצד מורכב של מרקטפלייסים.

מנוע מטצ'ינג בזמן אמת

הליבה של מרקטפלייס דו-צדדי. אלגוריתם מטצ'ינג מותאם, WebSocket לעדכונים, מערכת התראות חיה.

סליקה ישראלית מלאה

משולם / קארדקום / Tranzila. payouts לצד השני, refunds, חשבוניות אוטומטיות.

WhatsApp + SMS

Twilio / Vonage / Pelephone API. התראות אוטומטיות בנקודות הקריטיות של המסע.

פאנל ניהול מלא

סטטיסטיקות, ניהול משתמשים, ניהול עסקאות, KYC review, billing — לפי האפיון שלכם.

Database Schema מקצועי

PostgreSQL נורמלי + indexes, audit logs, RLS אמיתי. תשתית שמחזיקה ל-100x משתמשים.

למה לא Bubble

בכנות, על הסכנות של Bubble למרקטפלייס

חשוב לי לומר את זה ברור: יש פרויקטים ש-Bubble מעולה להם — landing pages, prototypes, פלטפורמות פנימיות פשוטות. מרקטפלייס דו-צדדי בזמן אמת עם סליקה — לא ביניהם. ראיתי יותר מדי פרויקטים נופלים בדיוק בנקודה הזו. הנה למה:

Bubble.io · מה שמחכה לכם

מציאות שיוצאת לאור אחרי 100 שעות
  • ✕ ביצועים — מתחת ל-1000 משתמשים נראה טוב. מעבר לזה הדפים נטענים ב-3-7 שניות. fix: aggressive caching, חבילות מחיר עולות.
  • ✕ Privacy Rules — מערכת שמורכבת לרוחב במהירות. במרקטפלייס דו-צדדי, שגיאת privacy = leak של מידע בין צד א' לצד ב'. ראיתי את זה קורה.
  • ✕ זמן אמת — אין WebSocket אמיתי. הכל polling שעולה כסף ועוד כסף.
  • ✕ סליקה ישראלית — אין integration native. plugins לא אמינים. תכננו לשבועות של דיבוג.
  • ✕ תמחור — workflow units אוזלים. שדרוג חבילה כל כמה חודשים. ברגע שהמרקטפלייס מצליח, החשבון מתפוצץ.
  • ✕ Vendor lock-in — לא ניתן לייצא את האפליקציה. אם Bubble נופלים / מעלים מחירים / שנהי מדיניות — אתם תקועים.
  • ✕ מובייל — responsive ב-Bubble בעייתי. PWA / native — לא ריאלי.
  • ✕ NDA / IP — הקוד לא באמת שלכם. ה-IP transfer הופך לתיאורטי.

קוד אמיתי + Claude Code

הגישה שלי
  • ✓ ביצועים — Next.js + Postgres = milliseconds. עומד באלפי משתמשים בו-זמנית כברירת מחדל.
  • ✓ אבטחה — authorization בקוד, RLS אמיתי ב-Postgres, audit logs. בנוי לסטנדרטים תאגידיים.
  • ✓ זמן אמת — WebSocket / SSE native. מטצ'ינג בזמן אמת אמיתי, לא סימולציה.
  • ✓ סליקה ישראלית — אינטגרציות עמוקות עם משולם / קארדקום / Tranzila. כמו במערכות שכבר בניתי.
  • ✓ תמחור — תשלמו hosting ($20-50/חודש) + DB. צומח לאט וצפוי.
  • ✓ בעלות — ה-repo שלכם, hosting שלכם. אין lock-in. תוכלו להחליף מפתח מחר אם תרצו.
  • ✓ מובייל — responsive ברמת פיקסל. PWA זמין. native app אפשרי.
  • ✓ NDA / IP — אמיתי. הקוד שלכם. ההעברה שלכם.

"אבל אתה כותב שזה לוקח אותו זמן — איך?"

בגלל ש-Claude Code מאפשר לי לכתוב קוד ב-velocity של no-code, אבל עם איכות production. 20+ שנות פיתוח שלי = יודע מה לבקש, מתי לעצור, איפה לבדוק. ההבדל בינ

הסטאק שאני בונה בו

Next.js (frontend), Node.js (backend), PostgreSQL (database), Stripe/משולם (payments), Twilio (SMS/WhatsApp), Vercel/Railway (hosting). הכל סטנדרט תעשייתי, מתועד, מבוקר.

למה זה משנה לכם

אם המרקטפלייס מצליח, רוצים שיצמח. לא רוצים מצב שכל פיצ'ר חדש דורש workaround. לא רוצים לראות את התקציב שלכם נשרף על Bubble subscription. רוצים לישון בלילה.

מה אם בכל זאת רוצים Bubble?

אני לא הצוות שלכם. אומר את זה בכנות — אני לא אבנה משהו שאני יודע שיגיע לקיר. יש מפתחי Bubble מעולים, ואני שמח להמליץ. אבל לא אעשה עבודה שאני יודע שתעלה לכם פי 3 בשנה הבאה.
