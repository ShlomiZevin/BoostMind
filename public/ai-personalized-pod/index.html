API ליצירת תמונות (Leonardo / Midjourney) Leonardo SDK ראשי, SDXL/Replicate fallback, IP-Adapter על תמונת ההתייחסות לעקביות דמות לאורך הספר. ✓
אופציונלי: וידאו / קול ElevenLabs לקול (קריינות בעברית/אנגלית), Runway/Pika לווידאו. מודולים מנותקים — מדליקים כשרוצים. ✓
אוטומציות Make / Zapier Make לאוטומציות "רכות" (מיילים, התראות, חיבורים פשוטים), קוד backend לזרימות הקריטיות. הגיבוי שלא נשבר. ✓

ניהול תהליך מלא: הזמנה → תוכן → הפקה → מסירה Job queue (BullMQ/Inngest) עם retry/backoff. כל שלב נכשל → מתוקן ללא איבוד הזמנה. שליטה מלאה ב-state. ✓
אינטגרציה ל-POD (Lulu / BookVault / וכד') חיבור REST API + webhooks. שליחת קובץ + פרטי הזמנה (SKU, כתובת), קבלת סטטוסים בזמן אמת. Lulu API מוכר אצלי. ✓
ניתוב אוטומטי: דיגיטלי → לקוח, מודפס → POD SKU-based routing במנוע ההזמנות. אם דיגיטלי — שליחת מייל עם לינק/קובץ. אם מודפס — POD API + מעקב. ✓
סטטוסים מהספק (ייצור/משלוח) ועדכון במערכת Webhook listener מאומת חתימה, עדכון DB, push ללקוח (מייל + WhatsApp/SMS אופציונלי). ✓
אזור ניהול בסיסי להזמנות Admin panel נקי — הזמנות, סטטוסים, תוכן AI שנוצר, אפשרות ל-retry ידני, אקספורט. קוד פתוח אצלך. ✓
תיעוד קצר README טכני + מדריך תפעולי קצר (PDF/Notion). איך מוסיפים ספק POD חדש, איך משנים פרומפט, איך מטפלים בשגיאה. ✓

הדבר היחיד שלא יהיה במודעה — ושצריך להיות

המודעה לא מזכירה בקרת איכות AI. זה הדבר שישבור את המערכת בעולם האמיתי: GPT יחזיר טקסט פוגעני, Leonardo יחזיר תמונה לא-מתאימה, ולקוח יקבל ספר שאי אפשר להראות לילד. אני בונה שכבת moderation אוטומטית (OpenAI Moderation + classifier מותאם) + ממשק human-in-the-loop למקרים גבוליים. זה אחד הדברים שמבחינים בין "אתר שעובד בדמו" ל"מערכת שבטוחה בהפעלה".

שבוע 5-6 · File Builder + תשלומים

הרכבת PDF + סליקה

תבנית PDF מוכנה להדפסה (300dpi, CMYK, bleeds), מנוע הרכבה, חיבור Stripe לתשלומים. קצה לקצה דיגיטלי: הזמנה → תשלום → PDF במייל.

שבוע 7-8 · POD Integration

חיבור Lulu API + ניתוב

שלב המפתח. חיבור REST + webhooks לספק POD, שליחת קובץ, מעקב סטטוסים, עדכון לקוח. בדיקות אמיתיות עם הזמנת test products.

שבוע 9-10 · אוטומציות + אדמין

Make/Zapier + פאנל ניהול

זרימות Make לאירועים "רכים" (מיילים, התראות), פאנל אדמין מלא, retry ידני, אקספורט, ניטור עלויות AI. בקרת איכות אוטומטית.

שבוע 11-12 · QA וההשקה

בדיקות עומסים + go-live

בדיקות end-to-end, עומסים, edge cases (ספק POD נופל, AI לא עונה, תשלום נכשל באמצע), תיעוד תפעולי, השקה רכה עם 5-10 הזמנות אמת.

על ספקי AI/POD — מודל העלות

ספקי AI (OpenAI, Leonardo) ו-POD (Lulu) מחייבים אותך ישירות. אני בונה לך גם דשבורד עלות לכל הזמנה כך שתדע בדיוק איפה אתה: לפי הניסיון שלי, ספר ילדים מותאם עם איורים = $1.50–$2.50 ב-AI cost לכל הזמנה. POD עצמו תלוי בספק (Lulu בערך $8–15 לספר A5 רך). הכל שקוף, ניתן למדידה, וניתן לאופטימיזציה לאחר ההשקה.
